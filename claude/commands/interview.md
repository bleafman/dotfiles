If a file path was provided ($1), read it first and use it as context.

Otherwise, ask me what I want to work on and let me describe it.

Then interview me in detail using the AskUserQuestion tool about literally anything: technical implementation, edge cases, scope, tradeoffs, UX, concerns, etc. Make sure the questions are not obvious - go deep and follow interesting threads.

Continue interviewing until it's complete, then offer to write the spec to a file.
